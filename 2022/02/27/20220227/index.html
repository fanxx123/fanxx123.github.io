ref属性

获取DOM元素，通过id名获取，Vue避免操作DOM ===> 使用ref


//标签添加ref属性
<h1 ref = 'title'></h1>
this.$refs.title
//在vc组件实例对象中获取真实DOM元素

//组件添加ref属性
<Zujian ref = 'name'/>
this.$refs.name
//获取的是Zujian的实例对象，而不是真实的DOM元素

props

父组件对子组件传参，子组件接受参数使用props

//父组件传参数
<Student name="张三" sex="男" :age="18"/>
//子组件props接收有三种写法
//第一种 ===> 简单接收
props:['参数1','参数2','参数3']

//第二种 ===> 限制类型
props:{
	参数: 类型
}

//第三种 ===> 限制类型 + 限制必要性 + 指定默认值
props:{
	参数: {
		type: 类型
		required: true
		default: 'xxx'
	}
}

注意：prop是单向数据流，不建议修改，因为没有_data，无法做数据代理。如果要修改，data里面接收一下。如果prop与data中的属性冲突，以prop的数据为主，prop的优先级高于data。

组件自定义事件

绑定自定义事件

父给子传参数 ===> props
子给父传参数 ===>

  1. 使用props，父给子提前传递一个回调函数，子组件调用函数时传递参数。
    //父组件
<School :getSchoolName = "getschoolname"/>
//父组件实例对象上配置methods
methods:{
 getschoolname(name){
  console.log('父组件收到了学校名：',name)
 }
}

//子组件
props: ['getschoolname']
methods: {
 sendschoolname(){
  //调用函数，传递参数
  this.getschoolname(this.name)
 }
}
  2. 使用自定义事件
    方法一：直接绑定自定义事件
    //父组件
//在父组件中给子组件绑定自定义事件，v-on在谁身上给谁的组件实例对象绑定自定义事件，需要在student组件上触发自定义事件
<Student v-on:zidingyi="getstudentname"><Student/>
//父组件中的回调函数
methods: {
 getstudentname(name){
  console.log(name)
 }
}

//子组件
//在子组件上触发自定义事件
methods: {
 chufazidingyi(){
  this.$emit("zidingyi",this.name)
 }
}
    方法二：灵活绑定自定义事件
    //父组件
<Student ref="student"><Student/>
//挂载完毕后绑定
mounted(){
 this.$refs.student.$on("zidingyi",this.getstudentname)
 //灵活绑定，如可实现等待几秒后绑定
 // this.$refs.student.$once('zidingyi',this.getstudentname) //绑定自定义事件（一次性）
}
methods: {
 getstudentname(name){
  console.log(name)
 }
}
//子组件同上

解绑自定义事件

在子组件中解绑自定义事件(触发事件$emit，解绑事件$off)

  • 解绑一个 this.$off("zidingyi1")
  • 解绑多个 this.$off(["zidingyi1","zidingyi2"])
  • 全部解绑 this.$off()
自定义事件使用时的两个问题
  • this的指向问题
    methods: {
 getstudentname(){
  console.log(this)
 }
}
this.$refs.student.$on("zidingyi",this.getstudentname)
//此时自定义事件给student组件绑定，所以，this指向的是student子组件
//但是这里不会出错，是因为与getstudentname里的this（父组件实例对象）发生了冲突，以gentstudentname中的this为主


this.$refs.student.$on("zidingyi",function(){
 console.log(this)
})
//这个this为子组件

this.$refs.student.$on("zidingyi",()=>{
 console.log(this)
})
//使用箭头函数，this指向父组件
  • 组件调用原生事件
    <Student v-on:click.native="getstudentname"><Student/>
<Student @click.native="getstudentname"><Student/>
//如果不加native，那么vue就不认识click指令，将它当作自定义指令来处理

全局事件总线

实现任意组件间通信。

原理

A组件想获得B组件的数据 ===>
在A组件中给总线（$bus）绑定自定义事件，回调函数留在A组件。
B组件触发自定义事件并传参

总线的特点

  1. 所有的组件都可以看见。
  2. 可以实现$on,$off,$emit方法。

使用什么可以让所有的组件都能看见？

一个重要的内置关系 —— VueComponent.prototype._proto === Vue.prototy
所以选择在Vue的原型对象上添加事件总线

//main.js文件中
new vm = new Vue({
   el: "#app",
   render: h => h(App),
   //安装全局事件总线
   beforeCreate(){
      Vue.prototype.$bus = this
      //this ——> vm实例对象，所有组件都可以访问vm
   }
})

//A组件，接收数据，绑定自定义事件
mounted(){
   this.$bus.$on("hello",(data)=>{
      console.log("收到B组件的数据："data)
   })
}

//B组件，传递数据，触发自定义事件
//触发
methods: {
   sendData(){
      this.$bus.$emit("hello",this.name)
   }
}
//B组件删除事件总线
beforeDestroy(){
   this.$bus.$off('hello')
}

消息订阅与发布

借助第三方库（pubsub-js）实现任意组件间通信，第三方库有很多，学习理念。
安装：npm i pubsub-js

原理

A组件：接收数据 ===> 订阅消息
B组件：传递数据 ===> 发布消息
A组件订阅消息，留一个回调函数。B组件发送消息并传递参数，调用A组件中的回调函数并接收参数。

//A组件
//引入第三方库
import pubsub from 'pubsub-js'
//订阅消息
mounted(){
   this.pubId = pubsub.subscribe("hello",function(){
      console.log("订阅成功")
   })
}
//A组件销毁前取消订阅
beforeDestroy(){
   pubsub.unsubscribe(this.pubId)
}

//B组件
//引入第三方库
import pubsub from 'pubsub-js'
//发布消息
methods: {
   publishNews(){
      pubsub.publish("hello",this.name)
   }
}
//subscribe方法的回调参数有两个
pubsub.subscribe("hello",function(msgName,data)){ }
//msgName: 消息名  data：数据

//subscribe方法的回调函数中this指向改变
pubsub.subscribe("hello",(msgName,data)=>{ })
//使用箭头函数解决this指向问题

$nextTick

Vue中的效率问题：回调全部执行，才解析模板
什么时候使用$nextTick:当改变数据后，要基于更新后的DOM进行某些操作时，要在$nextTick所指定的回调函数中执行。
$nextTick所指定的回调会在DOM节点更新完毕后执行。下一次DOM更新结束后执行其执行的回调。

this.$nextTick(function(){
   //下一轮执行
   //函数体
})

方法二：使用setTimeOut

setTimeOut(()=>{
   //异步执行
   //函数体
},200)

访问服务器，发送一个Ajax请求，请求方法：

  1. xhr
  2. 二次封装 jQuery $get $post

  4. fetch (windows内置方法,promise风格)
  5. vue-resourse
axios.get("http://localhost:5000/students").then(
   response =>{
      console.log(response.data)
   },
   error =>{
      console.log(error.msg)
   } 
)
//访问不到 http://localhost:5000，存在跨域问题

跨域限制
存在浏览器同源策略，是浏览器上为安全性考虑实施的非常重要的安全策略。

同源指的是什么

  1. 协议名一致
  2. 主机名一致
  3. 端口号一致
axios.get("http://localhost:5000/students").then(
   //访问：http://localhost:5000
   //本地：http://localhost:8080
   //协议名，主机名一致，端口号不相同，需要解决跨域问题
   response =>{
      console.log(response.data)
   },
   error =>{
      console.log(error.msg)
   }
)

如何解决跨域问题

  1. cors 特殊的响应头（非前端技术）
  2. jsonp 借助script标签里的src属性，在引入外部资源时不受同源策略的影响。（几乎不用，且只能解决get。问题面试问的多，开发用的少）
  3. 配置代理服务器

如何开启代理服务器

  1. nginx
  2. vue-cli

vue-cli开启代理服务器方法一：

//在vue.config.js文件中添加配置
devServer: {
   proxy: 'http://localhost:5000'
}
//axios发送请求
axios.get("http://localhost:8080/students").then(
   //向代理服务器请求信息
   //http://localhost:8080/ 相当于本地文件夹，students如果这个资源本地有，就不向5000端口发送请求
   response =>{
      console.log(response.data)
   },
   error =>{
      console.log(error.msg)
   }
)

//此方法存在的问题：
//1. 只能配置一个代理
//2. 不灵活控制走代理还是走本地

vue-cli开启代理服务器方法二：

devServer: {
   proxy: {
      '/api':{
         target: 'http://localhost:5000',
         pathRewrite: {'^/api':''},   //发送给5000服务器不带前缀
         ws: true,   //用于指示websocket
         changeOrigin: true   
         //控制请求头中的host值，跨域伪造，例如8080端口假装自己是5000端口，避免5000端口做限制。
      },
      '/api2':{
         target: 'http://localhost:5001',
         pathRewrite: {'^/api2':''},   //发送给5001服务器不带前缀
         ws: true,   //用于指示websocket
         changeOrigin: true   
         //控制请求头中的host值，跨域伪造，例如8080端口假装自己是5000端口，避免5000端口做限制。
      }
   }
}
//axios发送请求
axios.get("http://localhost:8080/api/students").then(
   //请求前缀添加在端口号后面，控制走不走代理，有前缀，走代理，没有前缀，走本地资源
   response =>{
      console.log(response.data)
   },
   error =>{
      console.log(error.msg)
   }
)
axios.get("http://localhost:8080/api2/students").then(
   response =>{
      console.log(response.data)
   },
   error =>{
      console.log(error.msg)
   }
)

插槽

  1. 作用：让父组件可以向子组件指定位置插入html结构，也是一种组件间通信的方式，适用于 父组件 ===> 子组件 。

  2. 分类：默认插槽、具名插槽、作用域插槽

  3. 使用方式：

    1. 默认插槽：

      父组件中：
        <Category>
           <div>html结构1</div>
        </Category>
子组件中：
        <template>
            <div>
               <!-- 定义插槽 -->
               <slot>插槽默认内容...</slot>
            </div>
        </template>

    2. 具名插槽：

      父组件中：
        <Category>
            <template slot="center">
              <div>html结构1</div>
            </template>

            <template v-slot:footer>
               <div>html结构2</div>
            </template>
        </Category>
子组件中：
        <template>
            <div>
               <!-- 定义插槽 -->
               <slot name="center">插槽默认内容...</slot>
               <slot name="footer">插槽默认内容...</slot>
            </div>
        </template>

    3. 作用域插槽：

      1. 理解：数据在组件的自身，但根据数据生成的结构需要组件的使用者来决定。（games数据在Category组件中，但使用数据所遍历出来的结构由App组件决定）

      2. 具体编码：

        父组件中：
		<Category>
			<template scope="scopeData">
				<!-- 生成的是ul列表 -->
				<ul>
					<li v-for="g in scopeData.games" :key="g">{{g}}</li>
				</ul>
			</template>
		</Category>

		<Category>
			<template slot-scope="scopeData">
				<!-- 生成的是h4标题 -->
				<h4 v-for="g in scopeData.games" :key="g">{{g}}</h4>
			</template>
		</Category>
子组件中：
        <template>
            <div>
                <slot :games="games"></slot>
            </div>
        </template>
		
        <script>
            export default {
                name:'Category',
                props:['title'],
                //数据在子组件自身
                data() {
                    return {
                        games:['红色警戒','穿越火线','劲舞团','超级玛丽']
                    }
                },
            }
        </script>
