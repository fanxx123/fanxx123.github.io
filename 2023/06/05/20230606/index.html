8.关系处理

1. 1:1关系

  • A表的一条记录只对应B表的一条记录

  • B表的一条记录 对应 A表的一条记录。

  • 关系处理：在B中添加一个A表的外键，且添加非空 不可重复两个约束。

  • ```sql
    用户和开户、老公和妻子
    –1:1 一个对象 对应 一个对象

      -- 外键+主键
  --主键的特点 = 唯一 不重复 不能为空
  --特殊的一对多关系

    drop table user_h cascade constraint;
    create table user_h(

       id number(10) primary key,
   name varchar2(20)

    );
    drop table balance cascade constraint;
    create table balance(

       userid number(10) references user_h(id) primary key,--unique not null
   balance number(10,2) check(balance>=0.0)

    );
    create table balance(

       id number(10) primary key,
   userid number(10) references user_h(id) unique not null,

## 2.`1:N`关系

+ `A`表的一条记录 对应 `B`表的`N`条记录。

+ `B`表的一条记录 对应 `A`表的一条记录。

+ 关系处理：在    多的一方 `B`   留   一得一方`A`      的一个外键。

  + 在`B`表中留`A`表的一个外键。

+ ```sql
  你和电脑、员工和部门
  -- 1:N   dept：emp 
     -- 一个部门可以有多个员工
     --但是一个员工只能有一个部门
     -- 在 N得一方(emp) 留 一的一方（dept） 的一个外键
  
  
  drop table dept cascade constraint;
  create table dept(
         id number(10) primary key,
         name varchar2(20) not null 
    );
  drop table emp cascade constraint;
  create table emp(
         id number(10) primary key,
         name varchar2(20) not null,
         deptid number(10) references dept(id) 
  );

3. N:N关系

  • A表中的一条记录对应B表中的N条记录

  • B表中的一条记录对应A表中的N条记录

  • 关系处理：将一个多对多关系拆分成两个1对多关系

  • 学生和课程
一个学生可以学习多门课程
一个课程可以被多个学生学习
学生：分数 = 1：N
课程：分数 = 1：N
学生     分数表     课程

在分数表中留 学生和课程的一个外键，附带联合主键。
